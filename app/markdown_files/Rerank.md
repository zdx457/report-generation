> 返回 [主文档 README.md](../README.md)

## 一、为什么需要 Rerank

向量检索（Bi-Encoder）速度快但精度有限，它只看"大致方向"是否接近。Rerank（Cross-Encoder）精度高但速度慢，适合对少量候选做精排。

|          | 向量检索 (Bi-Encoder)                  | Rerank (Cross-Encoder)                      |
| -------- | -------------------------------------- | ------------------------------------------- |
| **模型** | bge-m3                                 | Qwen3-VL-Reranker-8B                        |
| **原理** | 问题和文档各自独立编码，计算余弦相似度 | 问题和文档拼接输入，交叉注意力逐 token 比对 |
| **精度** | 粗筛，大致方向匹配                     | 精排，逐字语义匹配                          |
| **速度** | 极快（向量预计算，只需算问题向量）     | 较慢（每个文档都要完整过一遍模型）          |
| **适用** | 从大量数据中快速召回候选               | 从候选中精选最相关文档                      |

**设计逻辑**：向量检索负责"快而粗"地从大量数据中捞回候选，Rerank 负责"慢而精"地从候选中选出最相关的给 LLM。

## 二、Rerank 工作原理

### Cross-Encoder 交叉注意力机制

Rerank 将问题和每个候选文档拼接在一起，送入模型做交叉注意力计算：

```
                    ┌─────────────────────────┐
                    │   Qwen3-VL-Reranker-8B  │
                    │                         │
  "CT脑出血"  ──────►│  [CLS] CT 脑出血 [SEP]  │──────► relevance_score
                    │  检查类型：CT 诊断结论：  │        = 0.6881
  文档原文  ────────►│  脑出血 影像学表现：侧... │
                    │                         │
                    │   交叉注意力机制          │
                    │   每个token都能看到       │
                    │   问题和文档的所有token    │
                    └─────────────────────────┘
```

与 Bi-Encoder 的区别：Bi-Encoder 是问题和文档**分别**编码后算距离，Cross-Encoder 是**拼接**编码，每个 token 都能同时看到问题和文档的所有 token，因此精度更高。

### 实际示例

用户输入 **"动脉瘤（宽颈多发）"**，向量检索 top-5 后 Rerank 精排 top-3：

| Rerank排名 | 原始排名 | Rerank分数 | 诊断结论               |
| ---------- | -------- | ---------- | ---------------------- |
| 第1名      | Top-1    | 0.6881     | 动脉瘤（**宽颈多发**） |
| 第2名      | Top-2    | 0.4626     | 动脉瘤（多发）         |
| 第3名      | Top-5    | 0.1115     | 动脉瘤                 |

Rerank 正确地将"宽颈多发"排到了第1名，因为用户问题中明确包含了"宽颈"这个关键词，Cross-Encoder 能捕捉到这种细粒度匹配。

## 三、LLM 判断逻辑

Rerank 返回的 top-k 结果（带分数）会全部拼接进 Prompt 发给 LLM，由 LLM 依据分数选择最相关的参考进行结构化输出。

### Prompt 构造

```
参考信息：
【参考1】(Rerank相关性分数: 0.6881，来源: row22.md)
  诊断结论：动脉瘤（宽颈多发）
  影像学表现：右颈内动脉床突上段近后交通动脉起始部见分叶状动脉瘤影...

【参考2】(Rerank相关性分数: 0.4626，来源: row21.md)
  诊断结论：动脉瘤（多发）
  影像学表现：前交通动脉见囊袋状突起动脉瘤影...

【参考3】(Rerank相关性分数: 0.1115，来源: row380.md)
  诊断结论：动脉瘤
  影像学表现：双侧颈总动脉和颈内、颈外动脉形态和走行未见异常改变...

用户问题：动脉瘤（宽颈多发）
```

### LLM 判断规则（来自 prompt.md）

| 场景                  | LLM 怎么做                     | 示例                                                               |
| --------------------- | ------------------------------ | ------------------------------------------------------------------ |
| 分数最高且明显领先    | 直接用分数最高的参考输出       | 参考1(0.6881) >> 参考2(0.4626)，用参考1                            |
| 分数接近（差距<0.05） | 结合用户问题语义判断哪条更贴合 | 参考1(0.3334) ≈ 参考2(0.3333)，需结合问题语义                      |
| 分数接近但诊断不同    | 结合问题语义选最匹配的         | 参考1是"脑出血"，参考2是"脑出血破入脑室"，用户问"CT脑出血"→选参考1 |

### LLM 最终输出

LLM 只对选定的最相关参考做结构化输出，不会合并多条参考：

```markdown
## 一、影像学表现

右颈内动脉床突上段近后交通动脉起始部见分叶状动脉瘤影，瘤体突向右后下方，大小约4.9mm×7.2mm，瘤颈宽约4.7mm。

左后交通动脉起始部见动脉瘤影，瘤体突向左后下方，大小约4.2mm×2.7mm，瘤颈宽约2.5mm。

右大脑后动脉P1段见局限性囊袋状突起影，大小约1.8mm×1.8mm，颈部宽约1.8mm。

## 二、诊断意见

右颈内动脉床突上段近后交通动脉起始部宽颈动脉瘤；左后交通动脉起始部动脉瘤；右侧大脑后动脉P1段动脉瘤。
```

## 四、`rerank_top_k` 参数含义

`rerank_top_k` 控制 Rerank 精排后返回给 LLM 的候选数量：

| `rerank_top_k` | Rerank 评估数量 | 发给 LLM 的参考数 | 效果                                  |
| -------------- | --------------- | ----------------- | ------------------------------------- |
| 1              | 只评估1条       | 1条               | 最快，但 Rerank 没有对比空间          |
| 3              | 评估3条         | 3条               | Rerank 有对比空间，LLM 可参考分数判断 |
| 5              | 评估5条         | 5条               | 更多候选，但低分参考可能引入噪声      |

**注意**：`rerank_top_k` 不能超过 `top_k`（向量检索数量），代码中已做保护：

```python
if rerank_top_k > top_k:
    rerank_top_k = top_k
```

## 五、降级机制

当 Rerank API 调用失败时（网络超时、API Key 无效、服务不可用等），系统自动降级：

```python
try:
    rerank_results = rerank_documents(message, documents, top_n=rerank_top_k)
    # ... 正常 Rerank 流程
except Exception as e:
    # 降级：直接取向量检索的前 rerank_top_k 条
    reranked_entities = hit_entities[:rerank_top_k]
    for entity in reranked_entities:
        entity["_rerank_score"] = -1  # 标记为降级结果
```

降级后的行为：

- LLM 收到的参考信息不带 Rerank 分数（标记为 -1）
- 前端思考过程中会显示 ⚠️ 降级提示
- LLM 会按 prompt.md 中的规则，在无分数时直接依据语义判断

## 六、Rerank 模块接口

Rerank 功能已封装为独立模块 [rerank.py](../rerank.py)，供 `web.py` 和 `chat.py` 调用：

### `rerank_documents(query, documents, top_n=3)`

对候选文档进行 Rerank 精排。

**参数**：

- `query` (str)：用户查询文本
- `documents` (list[str])：候选文档文本列表
- `top_n` (int)：返回前 N 个最相关文档

**返回**：

- `list[dict]`：按 `relevance_score` 降序排列，每项包含：
  - `index`：原始 documents 列表中的索引
  - `relevance_score`：相关性分数（0~1）
  - `document`：`{"text": 原始文档文本}`

**用法**：

```python
from rerank import rerank_documents, get_rerank_config

results = rerank_documents("CT脑出血", ["文档1", "文档2", "文档3"], top_n=2)
for r in results:
    print(f"排名: {r['index']}, 分数: {r['relevance_score']:.4f}")

config = get_rerank_config()
print(f"模型: {config['rerank_model']}, API Key: {'已配置' if config['api_key_configured'] else '未配置'}")
```

## 七、评估结果

使用 [eval_rerank_compare.py](../test/eval_rerank_compare.py) 对 50 条样本的评估结果：

| 指标         | 向量检索 | 向量检索+Rerank | 变化      |
| ------------ | -------- | --------------- | --------- |
| Top-1 命中率 | 78.0%    | 80.0%           | **+2.0%** |
| Top-k 命中率 | 98.0%    | 96.0%           | -2.0%     |
| MRR          | 0.8657   | 0.8767          | +0.011    |

- Rerank 改善了 7 条（向量检索未命中但 Rerank 命中）
- Rerank 退化了 6 条（向量检索命中但 Rerank 未命中）
- 净提升 +1 条

退化案例的特点是 Rerank 分数非常接近（如 0.8333 vs 0.8310），说明这些情况下 Rerank 模型也不太确定哪条更相关，这正好是 LLM 利用分数做判断的场景。
