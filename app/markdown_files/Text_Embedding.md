## 一、数据入库流程（上传文件）

### Step 1: 保存原始文件

```
用户在 Web 界面上传 xlsx 文件（支持多文件）
    │
    ▼
xlsx 文件保存到 report_template/ 目录
（自动跳过 ~$ 开头的临时文件和非 xlsx 文件）
```

**对应代码：** `web.py` → `upload_and_process()`

**示例：**

```
上传: 绥化_报告_模版2.xlsx
保存: report_template/绥化_报告_模版2.xlsx
```

### Step 2: 按行切片

```
读取 xlsx 第一行作为表头
每一行数据 → 一个 Markdown 文件
保存到 xlsx_slices/ 目录
```

**切片后的 md 文件格式：**

```markdown
## 绥化*报告*模版2

| 检查类型 | 部位 | 检查项目 | 诊断结论 | 影像学表现                    | 影像学意见         | 来源up_id | 来源id  | 模态/部位/检查       |
| -------- | ---- | -------- | -------- | ----------------------------- | ------------------ | --------- | ------- | -------------------- |
| CT       | 头颅 | 颅脑平扫 | 脑出血   | 侧基底节区见团块状高密度影... | 侧基底节区脑出血。 | 3000017   | 3000073 | CT / 头颅 / 颅脑平扫 |
```

**命名规则：** `{xlsx文件名}_row{行号}.md`，例如 `绥化_报告_模版2_row1.md`

**对应代码：** `web.py` → `_xlsx_to_slices()` + `_slice_to_md()`

### Step 3: 增量检查

```
查询 Milvus 已有的 source 集合
对比新切片的文件名
过滤掉已入库的切片，只处理新增的
```

**为什么需要增量检查？** 避免重复向量化，节省 API 调用和时间。同一个 xlsx 多次上传不会产生重复数据。

**对应代码：** `web.py` → `upload_and_process()` 中的增量检查逻辑

### Step 4: 解析 md → 自然语言文本

```
提取 md 文件中的表头和数据行
将每个字段拼接为自然语言文本
同时提取元数据
```

**拼接规则：** `字段名：值`，用换行符连接

**拼接后的文本（用于向量化）：**

```
检查类型：CT
部位：头颅
检查项目：颅脑平扫
诊断结论：脑出血
影像学表现：侧基底节区见团块状高密度影，CT值约61HU，大小约2.8cm×3.0cm，灶周见低密度水肿带，侧侧脑室受压，三脑室受压变形，中线结构向对侧移位。脑干及小脑未见异常。
影像学意见：侧基底节区脑出血。
来源up_id：3000017
来源id：3000073
模态/部位/检查：CT / 头颅 / 颅脑平扫
```

**元数据（单独存储，支持过滤查询）：**

| 字段     | 值       |
| -------- | -------- |
| 检查类型 | CT       |
| 部位     | 头颅     |
| 检查项目 | 颅脑平扫 |
| 诊断结论 | 脑出血   |

**对应代码：** `web.py` → `_parse_md_slice()`

### Step 5: 批量向量化

```
调用 bge-m3 Embedding API
每 16 条为一批次
失败自动重试 5 次（间隔递增：5s, 10s, 15s, 20s）
每条文本 → 1024 维向量
```

**API 请求格式：**

```json
{
  "model": "bge-m3",
  "input": ["检查类型：CT\n部位：头颅\n...", "检查类型：MRI\n..."]
}
```

**对应代码：** `web.py` → `_get_embeddings_batch()`

### Step 6: 写入 Milvus Lite

```
每条记录包含向量、文本、元数据
每 500 条批量插入
自动创建集合（如不存在）
```

**每条记录结构：**

| 字段     | 类型        | 示例                            |
| -------- | ----------- | ------------------------------- |
| vector   | FLOAT[1024] | [0.023, -0.045, ...]            |
| text     | VARCHAR     | "检查类型：CT\n部位：头颅\n..." |
| source   | VARCHAR     | "绥化*报告*模版2_row9.md"       |
| 检查类型 | VARCHAR     | "CT"                            |
| 部位     | VARCHAR     | "头颅"                          |
| 检查项目 | VARCHAR     | "颅脑平扫"                      |
| 诊断结论 | VARCHAR     | "脑出血"                        |

**对应代码：** `web.py` → `upload_and_process()` 中的插入逻辑

## 二、报告生成流程

### Step 1: 向量检索

```
用户输入: "CT脑出血"
    │
    ▼
bge-m3 向量化 → [0.12, -0.03, 0.08, ...] (1024维)
    │
    ▼
Milvus 余弦相似度检索 → 返回 Top-K 候选（默认5条）
```

**检索结果示例：**

| 排名  | 相似度 | 诊断结论             | 来源    |
| ----- | ------ | -------------------- | ------- |
| Top-1 | 0.3249 | 脑出血（破入脑室）   | row10   |
| Top-2 | 0.3262 | 脑出血-CT颅脑平扫    | row2131 |
| Top-3 | 0.3273 | 脑出血               | row374  |
| Top-4 | 0.3326 | 脑出血               | row9    |
| Top-5 | 0.3389 | 脑出血并破入脑室系统 | row2037 |

**对应代码：** `web.py` → `get_embedding()` + `search()`

### Step 2: Rerank 精排

```
将 Top-K 候选文本和用户问题一起送入 Reranker
Cross-Encoder 逐 token 交叉比对
按 relevance_score 重新排序
取 Top-N（默认3条），每条带相关性分数
```

**Rerank 原理（Cross-Encoder）：**

```
                    ┌─────────────────────────┐
                    │   Qwen3-VL-Reranker-8B  │
                    │                         │
  "CT脑出血"  ──────►│  [CLS] CT 脑出血 [SEP]  │──────► relevance_score
                    │  检查类型：CT 诊断结论：  │        = 0.3328
  文档原文  ────────►│  脑出血 影像学表现：侧... │
                    │                         │
                    │   交叉注意力机制          │
                    │   每个token都能看到       │
                    │   问题和文档的所有token    │
                    └─────────────────────────┘
```

**Rerank 结果示例（top-3）：**

| Rerank 排名 | 原始位置 | Rerank 分数 | 诊断结论           |
| ----------- | -------- | ----------- | ------------------ |
| 第1名       | Top-4    | 0.3328      | 脑出血             |
| 第2名       | Top-3    | 0.3279      | 脑出血             |
| 第3名       | Top-1    | 0.3193      | 脑出血（破入脑室） |

**为什么 Rerank 把 Top-4 提到了第1名？**

- Top-1 "脑出血（破入脑室）" 虽然向量相似度高，但"破入脑室"是更具体的子类型
- Top-4 "脑出血" 与 "CT脑出血" 查询最精确匹配
- Reranker 能理解这种细微差别，向量检索只看整体方向

> 📖 Rerank 原理、LLM 判断逻辑、降级机制等详见 [Rerank.md](Rerank.md)

**⚠️ 降级机制：** Rerank 服务不可用时，自动降级为直接使用向量检索的 Top-N 结果，不影响正常使用。

**对应代码：** `rerank.py` → `rerank_documents()`

```

```

**LLM 判断逻辑：** LLM 依据 Rerank 分数选择最相关参考，只对最相关参考做结构化输出。详见 [Rerank.md](Rerank.md#三llm-判断逻辑)

**对应代码：** `web.py` → `rag_respond()` 中的 Prompt 拼接逻辑

### Step 4: LLM 判断 + 流式生成

```
Qwen 模型接收 messages（包含 Rerank 分数）
依据 prompt.md 规则判断：按 Rerank 分数选最相关参考
只对最相关参考做结构化输出（影像学表现 + 诊断意见）
流式返回给前端展示
```

**LLM 判断过程：**

```
1. 看Rerank分数 → 参考1分数最高(0.3328)
2. 检查分数差距 → 参考1(0.3328)和参考2(0.3279)差距<0.05，分数接近
3. 结合问题语义 → 用户问"CT脑出血"，参考1和参考2都是"脑出血"
4. 选择最匹配的 → 参考1最贴合
5. 只对参考1做结构化输出
```

**输出格式：**

```markdown
## 一、影像学表现

侧基底节区见团块状高密度影，CT值约61HU，大小约2.8cm×3.0cm。

灶周见低密度水肿带，侧侧脑室受压，三脑室受压变形，中线结构向对侧移位。

脑干及小脑未见异常。

## 二、诊断意见

侧基底节区脑出血。
```

## 向量检索 vs Rerank 对比

|          | 向量检索 (Bi-Encoder)                  | Rerank (Cross-Encoder)                      |
| -------- | -------------------------------------- | ------------------------------------------- |
| **模型** | bge-m3                                 | Qwen3-VL-Reranker-8B                        |
| **原理** | 问题和文档各自独立编码，计算余弦相似度 | 问题和文档拼接输入，交叉注意力逐 token 比对 |
| **精度** | 粗筛，大致方向匹配                     | 精排，逐字语义匹配                          |
| **速度** | 极快（向量预计算）                     | 较慢（每对实时计算）                        |
| **适用** | 从大量数据中快速召回候选               | 从候选中精选最相关文档                      |

**设计逻辑：** 向量检索负责"快而粗"地从大量数据中捞回候选，Rerank 负责"慢而精"地从候选中选出最相关的给 LLM。

> 📖 更多 Rerank 原理、LLM 判断逻辑、参数含义、降级机制详见 [Rerank.md](Rerank.md)
