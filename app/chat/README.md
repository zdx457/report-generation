# Chat 模块 — ReAct 多轮推理对话

基于 `memory` 模块的短期/长期记忆，实现 **ReAct 循环推理对话**，支持流式输出。

提供两种模式：

- **纯推理模式**：不依赖检索，仅靠 LLM 多步推理
- **带检索模式**：集成 RAG 多路召回 + Rerank，可调用报告数据库检索

---

## 模块结构

```
chat/
├── chat.py              # ReAct 纯推理对话终端（可独立运行）
├── rag_chat.py          # ReAct + RAG 检索对话终端（可独立运行）
└── README.md            # 本文件
```

## 设计决策

| 决策项   | 选择                                      | 说明                                                              |
| -------- | ----------------------------------------- | ----------------------------------------------------------------- |
| 推理模式 | ReAct 循环                                | LLM 每轮输出推理/动作 → 系统执行（检索） → LLM 看到结果再判断     |
| 输出格式 | `[CONTINUE]`/`[ACTION: search]`/`[FINAL]` | 简单标记，LLM 容易遵守，代码容易解析                              |
| 输出方式 | 流式 SSE                                  | 边生成边显示，用户体验好，不用等完整生成                          |
| 终止条件 | LLM 自主判断                              | 每轮把之前所有推理/检索结果展示给 LLM，让它自己决定"够了没"       |
| 最大步数 | 5 步                                      | 防止无限循环，超限后强制基于已有推理输出最终回答                  |
| 记忆集成 | 依赖 memory 模块                          | 继承 STM 的智能摘要、上下文消解、实体追踪；LTM 的时间衰减偏好统计 |
| RAG 检索 | 复用现有管线                              | 复用 `retrieval.py` + `rerank.py`，保持架构一致                   |

## 核心功能

### 1. 纯推理模式（chat.py）

```
用户输入: "脑出血和高血压有什么关系"
    │
    ▼
[第1步] LLM 输出 → [CONTINUE] 推理内容1
    💭 流式显示给用户
    │
    ▼
[第2步] LLM 看到"你之前的推理：第1步..." → 判断 → [CONTINUE] 推理内容2
    💭 流式显示给用户
    │
    ▼
[第3步] LLM 看到"你之前的推理：第1步、第2步..." → 判断 → [FINAL] 最终回答
    流式显示给用户
    │
    ▼
写入 STM + LTM
```

### 2. 带检索模式（rag_chat.py）

```
用户输入: "左基底节区高密度影和高血压有关系吗"
    │
    ▼
[第1步] LLM 判断需要检索 → [ACTION: search] 左基底节区 高密度影 高血压
    🔍 执行检索：向量多路召回 + Rerank → 返回参考报告
    │
    ▼
[第2步] LLM 看到检索结果 → [CONTINUE] 分析检索结果
    💭 流式显示推理过程
    │
    ▼
[第3步] LLM 综合推理 → [FINAL] 最终回答
    流式显示给用户
    │
    ▼
写入 STM + LTM
```

### 3. 流式输出

- 使用 SSE（Server-Sent Events）协议，`stream: True`
- `iter_lines` 逐块解析，`print(content, end="", flush=True)` 边生成边显示
- 推理步骤带 `💭 [第X步]` 前缀，检索操作显示 `🔍 检索: xxx`，最终回答带 `\nAI:` 前缀

### 4. 记忆集成

- **短期记忆**：上下文消解（补充缺省实体）、智能摘要（淘汰轮次压缩）
- **长期记忆**：偏好统计（时间衰减加权）、会话生命周期管理

## 工作流对比

| 阶段 | 纯推理模式 `chat.py` | 带检索模式 `rag_chat.py` |
| ---- | -------------------- | ------------------------ |
| 初始 | `[CONTINUE]` 推理    | `[ACTION: search]` 检索  |
| 推理 | 每步展示推理给用户   | 每步展示推理给用户       |
| 工具 | 无                   | RAG 检索自动执行         |
| 终止 | LLM 输出 `[FINAL]`   | LLM 输出 `[FINAL]`       |

## 内置命令

两种模式都支持以下命令：

| 命令        | 说明                       |
| ----------- | -------------------------- |
| `exit/quit` | 退出程序，自动保存长期记忆 |
| `clear`     | 清空当前会话的短期记忆     |
| `info`      | 查看短期/长期记忆状态      |
| 直接输入    | 进入 ReAct 推理循环        |

## 使用方式

### 运行纯推理模式

```bash
cd app/chat
python chat.py           # 普通模式，每步显示推理过程
python chat.py --debug   # 调试模式，显示 LLM 输入输出细节
```

### 运行带检索模式

```bash
cd app/chat
python rag_chat.py           # 普通模式，支持自动检索
python rag_chat.py --debug   # 调试模式，显示检索结果和 messages
```

要求：

- `milvus_lite.db` 必须存在（在 `app/` 目录下）
- `.env` 中配置 `EMBED_URL`, `CHAT_URL`, `CHAT_MODEL` 等环境变量
